✅ GOOD NEWS: All scripts use RELATIVE PATHS or DYNAMIC PATH DETECTION
   You DON'T need to change any paths when deploying to a new laptop!

================================================================================
HOW IT WORKS:
================================================================================

All Scripts Use:
---------------
✅ __dirname (Node.js) - Automatically gets current directory
✅ path.resolve(__dirname, 'server.js') - Relative path, works anywhere
✅ $MyInvocation.MyCommand.Path (PowerShell) - Gets script location automatically
✅ Split-Path -Parent - Gets directory dynamically

Examples:
---------
- install-service.js uses: path.resolve(__dirname, 'server.js')
- server.js uses: __dirname and relative paths
- All PowerShell scripts use: $scriptDir = Split-Path -Parent $MyInvocation.MyCommand.Path

This means:
----------
✅ Works on ANY path on ANY laptop
✅ No path changes needed
✅ No configuration file editing needed

================================================================================
WHEN YOU INSTALL ON NEW LAPTOP:
================================================================================

1. Copy PrintServer folder to ANY location
   Example: C:\PrintServer
   OR: D:\MyApp\PrintServer
   OR: E:\Software\PrintServer
   ANY location works!

2. Run: npm install

3. Run: SETUP-NEW-LAPTOP.ps1 (as Administrator)

4. Done! Service will be installed with correct paths automatically
